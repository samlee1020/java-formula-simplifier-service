# 输入格式说明

本文档根据当前代码实现反推，主要依据 `Input.java`、`Definer.java`、`Lexer.java`、`Parser.java` 和相关 `Factor` 类。

程序从标准输入依次读取：

```text
普通函数个数 m
普通函数定义 1
...
普通函数定义 m
递推函数个数 r
递推函数定义 1 的第 1 行
递推函数定义 1 的第 2 行
递推函数定义 1 的第 3 行
...
递推函数定义 r 的第 1 行
递推函数定义 r 的第 2 行
递推函数定义 r 的第 3 行
目标表达式
```

每一行输入都会删除所有空白字符，所以空格、制表符不影响解析。

## 普通函数

普通函数每个占 1 行，形式为：

```text
f(x,y)=表达式
```

其中：

- 函数名只能是 `f`、`g`、`h` 之一。
- 形参写在小括号内，用英文逗号分隔。
- 形参通常使用 `x`、`y`、`z`。
- 形参顺序不固定，函数调用时按位置绑定实参。例如 `f(y,x)=y-x` 调用 `f(2,x)` 会把 `y=2`、`x=x`。
- 普通函数在内部会被当作下标为 `0` 的函数保存，因此调用 `f(...)` 等价于调用该函数的第 0 项定义。

示例：

```text
f(x,y)=x+y
g(x)=sin(x)+x^2
```

调用示例：

```text
f(x,2)
g(x)
```

## 递推函数

递推函数每个占 3 行，必须依次给出第 0 项、第 1 项和递推关系：

```text
f{0}(x,y)=表达式0
f{1}(x,y)=表达式1
f{n}(x,y)=递推表达式
```

其中：

- 函数名同样只能是 `f`、`g`、`h`。
- 形参可以有多个，建议使用 `x`、`y`、`z`，例如 `g{0}(x,y)=x+y`。
- 形参顺序不固定，可以写成 `g{0}(y,x)=...`。
- 同一个递推函数三行的形参列表和顺序必须保持一致，例如都写 `g{0}(y,x)`、`g{1}(y,x)`、`g{n}(y,x)`。
- 当前实现只从第 0 行读取形参列表，不会校验第 1 行和第 n 行是否一致；调用方应主动保证一致性。
- 第 1 行和第 2 行分别提供 `0`、`1` 两个初值。
- 第 3 行提供 `n` 项递推关系。
- 递推关系里主要支持用 `n-1`、`n-2` 调用前两项，例如 `f{n-1}(x,y)`、`f{n-2}(x,y)`。
- 实际调用时需要写具体非负整数下标，并传入对应数量的实参，例如 `f{3}(x)`、`g{2}(x,2)`。
- 多形参递推函数可以展开和计算，但 `y`、`z` 应作为函数定义里的形参占位符使用，不作为目标表达式自由变量。

示例：

```text
f{0}(x)=x
f{1}(x)=x^2
f{n}(x)=f{n-1}(x)+f{n-2}(x)
```

调用：

```text
f{3}(x)
```

会递归展开为：

```text
f{2}(x)+f{1}(x)
```

再继续展开。

多形参示例：

```text
g{0}(x,y)=x+y
g{1}(x,y)=x*y
g{n}(x,y)=g{n-1}(x,y)+g{n-2}(x,y)
```

调用：

```text
g{2}(x,2)
```

输出：

```text
3*x+2
```

## 目标表达式语法

目标表达式支持：

- 整数常量：`0`、`1`、`123`。
- 变量 token：面向前端产品时只开放自由变量 `x`；`y`、`z` 只用于函数定义形参。
- 加减乘：`+`、`-`、`*`。
- 乘方：`^`，指数写整数，例如 `x^2`、`(x+1)^3`。
- 括号表达式：`(表达式)`。
- 三角函数：`sin(因子)`、`cos(因子)`，可带指数，例如 `sin(x)^2`。
- 自定义普通函数调用：`f(实参列表)`、`g(...)`、`h(...)`。
- 自定义递推函数调用：`f{整数}(实参列表)`。
- 求导：`dx(表达式)`。

表达式的大致结构为：

```text
表达式 = [+-] 项 { (+|-) 项 }
项     = 因子 { * 因子 }
因子   = [+-] 数字
       | 变量 [^整数]
       | (表达式) [^整数]
       | sin(因子) [^整数]
       | cos(因子) [^整数]
       | 函数调用
       | dx(表达式)
```

函数调用的实参在代码中按“因子”解析。如果要把一个完整表达式作为实参，需要用括号包起来：

```text
f((x+1),2)
```

而不是：

```text
f(x+1,2)
```

## 完整输入示例

示例 1：普通函数

```text
1
f(x,y)=x+y
0
f(x,2)
```

输出：

```text
x+2
```

示例 2：递推函数

```text
0
1
f{0}(x)=x
f{1}(x)=x^2
f{n}(x)=f{n-1}(x)+f{n-2}(x)
f{3}(x)
```

输出：

```text
2*x^2+x
```

示例 3：求导

```text
0
0
dx(x^2+sin(x))
```

输出：

```text
2*x+cos(x)
```

## 注意事项

- 输入解析几乎没有错误处理，格式不符合预期时可能直接抛异常或得到错误结果。
- 函数名通过定义行第一个字符取得，因此定义行必须以 `f`、`g` 或 `h` 开头。
- 递推函数定义通过第 3 个字符判断 `{0}`、`{1}`、`{n}`，所以推荐严格写成 `f{0}(...)`、`f{1}(...)`、`f{n}(...)`。
- 递推展开只替换字符串 `n-1` 和 `n-2`，不支持更一般的 `n-3`、`n+1`、单独的 `n` 等下标表达式。
- 词法器能识别 `x`、`y`、`z`，但内部多项式结构只保存一个变量指数，最终都会按 `x` 输出。也就是说它不是完整的多元多项式化简器。
- 因为上述限制，前端目标表达式不要允许自由 `y`、`z`；函数调用实参建议只使用整数、`x`、或由 `x` 构成的括号表达式。
- 函数参数替换是字符串替换，形参名最好只使用单字符 `x`、`y`、`z`，避免和函数体里的其它文本意外重名。
- 隐式乘法不支持，必须写 `2*x`，不能写 `2x`。
- 三角函数括号里解析的是一个因子。若要写完整表达式，也需要额外括号，例如 `sin((x+1))`。
- `dx` 的实现是看到字符 `d` 后跳过下一个字符，因此实际应写成 `dx(...)`。
